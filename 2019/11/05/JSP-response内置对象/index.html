    response 对象是由 Tomcat 创建的，代表服务器对客户端的响应。它的继承体系和 request 对象是完全对应的，这里就不再总结了。当一次 HTTP 请求结束的时候，response 对象就会返回响应报文给客户端浏览器，浏览器收到响应报文后，就会按照响应报文的要求执行相关命令。

    response 对象仅仅是设定了返回报文的控制信息而已，只有响应报文送到浏览器，由浏览器解析执行后才会真正有用。理解这一点至关重要！

    通过 response 对象设定请求头信息

    HttpServletResponse 对象提供了void setHeader(String name,String value)方法来设置响应头信息。有以下两种典型使用方式：

    • 通过response.setHeader("refresh","1")方法设定响应报文的响应头信息，该方法要求浏览器 1 秒钟以后重新发送一次请求到当前页面。

    • 通过response.setHeader("refresh","2;URL=responseDemo02.jsp")方法，指定浏览器 2 秒钟后重新发送请求到指定的 url 中。这里的第二个参数使用方式有点怪，内部肯定是做了字符串解析的。什么时候看下源码？
